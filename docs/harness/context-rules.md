# コンテキスト管理ルール

## コンテキスト規律

- `node_modules/` や `.next/` を grep 対象に含めない
- 大きな法令データ・JSONファイルは必要な部分だけ読む
- セッション開始時: `CLAUDE.md` → `AGENTS.md` → 関係する `docs/specs/` の順で読む

---

## 状態の明示化

- 長時間タスクはチェックポイントを明示する（例: Step 1完了 → Step 2へ）
- 中断時は「何が完了していて何が残っているか」を `docs/exec-plans/active/` に記録してから止める

---

## リポジトリ外の知識は存在しない

Slackの議論・口頭での合意・チャットの決定事項はエージェントには見えない。
アーキテクチャの判断・設計の意図・法令解釈の合意はすべてリポジトリ内のMarkdownとしてエンコードする。
