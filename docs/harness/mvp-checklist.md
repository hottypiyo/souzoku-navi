# MVP思考チェックリスト（相続ナビ版）

新機能・新ページを実装する前に、以下の問いに答えてから着手すること。

## 実装前4問

1. **誰が**: 相続人のどのセグメント？
   - 被相続人の子 / 配偶者 / 兄弟
   - 専門家委任済み / DIY
   - 手続きの緊急度（期限切迫 / 余裕あり）

2. **どんな状況で**:
   - 死亡直後の手続き混乱期か
   - 落ち着いた調査フェーズか
   - 書類作成直前か

3. **どんな課題を**: 「ナビできる」では不十分。
   - 何の手続きが必要か分からない
   - 順序が分からない
   - 必要書類が分からない
   - どこに相談すべきか分からない
   ——のどれか

4. **データ設計で**:
   - 被相続人情報・相続人構成・財産情報をどこまで保持するか
   - ユーザーをまたいで共有するか（家族間コラボ）
   - 法改正に追随する必要がある条文・数値はどこに持つか

**答えられない問がある場合、`docs/specs/` にメモしてから着手する。**

---

## 仕様駆動（Spec-First）

- 新機能は `docs/specs/<feature>.md` に仕様書を先に作成する
- 仕様書には「スコープ外（今回作らないもの）」と「データ設計の仮定」を必ず記載する
- 法改正の影響を受けうる数値・条文は仕様書でフラグを立てる
- スペックテンプレート: `/spec-draft` コマンドを使う
