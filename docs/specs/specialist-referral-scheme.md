# 専門家紹介スキーム — 法的設計仕様書

作成日: 2026-04-30  
担当: CLO先行設計 → CTO実装

---

## 1. ビジネスモデルの前提

- **月額サブスクは収支トントンのフィルター**: 真剣に手続きを進める優良顧客を識別する機能
- **収益の本丸は専門家紹介**: 相続案件は高単価（遺産整理業務100万〜300万円）のため、1件成約で大きな収益が生まれる
- **競争優位**: 手続き進行状況・財産情報を把握しているため、「今まさに専門家が必要なタイミング」を機械的に検知できる

---

## 2. 法的リスクの整理

### 2.1 弁護士法72条（非弁行為）

> 弁護士でない者が、報酬を得る目的で、訴訟事件、非訟事件及び審査請求、再審査請求等行政庁に対する不服申立事件その他一般の法律事件に関して法律事務を取り扱い、又はこれらの周旋をすることを業とすること

**「周旋」に該当するか否かが最大リスク**。  
判例上、単なる情報提供・紹介（利用者が自分で連絡する）は周旋に当たらないとされるが、  
「報酬を得て、特定の弁護士への斡旋を業として行う」と判断されると72条違反となりうる。

### 2.2 司法書士法・行政書士法

弁護士法72条と同様の非業務制限規定あり。同じ論理が適用される。

### 2.3 税理士法52条

> 税理士又は税理士法人でない者は、この法律に別段の定めがある場合を除くほか、税理士業務を行ってはならない。

税務相談・申告代行を自社で行うことは不可。紹介に留める限りは問題なし。

## 3. 安全な収益化モデルの選択肢

### オプションA: 情報掲載モデル（最も安全）

専門家が自社の情報をディレクトリに掲載する対価として月額/年額掲載料を徴収。  
利用者は掲載情報を見て自分でコンタクトする。

- **法的根拠**: 広告業務に相当。周旋に当たらない。
- **リスク**: 低
- **収益**: 掲載料 × 専門家数（スケールに時間がかかる）
- **実装難易度**: 低

### オプションB: 成果報酬モデル（要注意）

利用者が専門家と成約した場合に、専門家から紹介手数料を受け取る。

- **法的根拠**: 「周旋の業」に当たる可能性あり（弁護士法72条違反リスク）
- **リスク**: 高（弁護士については特に）
- **対策**: 弁護士ドットコム等は「相談の場の提供」として72条をクリアしている設計
- **実装難易度**: 高（契約設計・法的レビュー必須）

### オプションC: スポンサーシップ/広告モデル

「このタスクには税理士が必要です」という文脈で、スポンサー専門家を表示する。  
CPC（クリック課金）またはCPM（表示課金）で徴収。

- **法的根拠**: 広告掲載業務。周旋に当たらない。
- **リスク**: 低〜中（表示内容が「推薦」として受け取られないよう「広告」表示が必要）
- **収益**: タスク閲覧数 × CTR × CPC
- **実装難易度**: 中

## 4. 推奨スキーム（フェーズ設計）

### Phase 1（現在〜利用者100名まで）: **情報掲載モデル**

- 専門家ディレクトリ掲載：無料（関係構築・実績収集）
- 利用者は掲載情報を見てコンタクト
- 自社は「紹介した」という関係を専門家と構築する

### Phase 2（利用者100〜500名）: **掲載料 + 文脈広告**

- 専門家掲載料: 月額3,000〜5,000円/件
- 特定タスク（相続税申告、不動産登記等）にスポンサー枠表示
- 「PR」「広告」を明示することで景品表示法・広告規制をクリア

### Phase 3（利用者500名超）: **法的スキームを整備して成果報酬へ移行を検討**

- 顧問弁護士による契約スキームのレビュー
- 「専門家とのマッチング」ではなく「コンサルティングの場の提供」として設計
- 弁護士については72条の解釈を確認後に実施

## 5. 開示・表示要件

### 必須表示

```
本サービスは情報提供を目的としており、法律相談・税務相談ではありません。
専門家の選定・契約はお客様ご自身の判断と責任において行ってください。
掲載している専門家の業務品質について、当社は保証いたしません。
```

### 広告枠の場合

```
[PR] / [広告] / [スポンサード]
```

の明示が景品表示法上必要。

## 6. 専門家審査基準（掲載承認の要件）

| 項目 | 基準 |
|---|---|
| 資格 | 各士業の登録番号を確認（弁護士会・司法書士会・税理士会） |
| 専門領域 | 相続業務の取り扱い実績あり（自己申告+ヒアリング） |
| 地域 | 日本国内 |
| 承認フロー | 管理者が `is_approved` フラグを手動で `true` に設定 |

---

## 7. 実装上の注意点

- 専門家ページに「この事務所に相談する」ボタンを設置する場合、ボタンは **外部リンク or メール** にする
- 自社システム上での「相談申込」フォームは設置しない（周旋に当たるリスク）
- 成約確認・追跡の仕組みはPhase 3まで実装しない
- 掲載料の請求は Stripe で行う（専門家向けサブスク）
